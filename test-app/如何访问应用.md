# 🌐 如何访问 test-app

## 📋 构建 vs 运行

- **构建** (`npm run build`): 只是生成优化的生产文件，不会启动服务器
- **运行**: 需要启动服务器才能通过浏览器访问

## ✅ 两种访问方式

### 方法 1: 开发模式（推荐）

**特点：**
- ✅ 支持热重载（修改代码自动刷新）
- ✅ 有详细的错误提示
- ✅ 适合开发和调试

**启动命令：**
```powershell
# 在根目录
npm run dev:test-app

# 或者进入目录
cd test-app
npm run dev
```

**访问：**
- 打开浏览器访问：http://localhost:3000
- 如果 3000 端口被占用，Next.js 会自动使用 3001、3002 等
- 查看终端输出确认实际端口

### 方法 2: 生产模式

**特点：**
- ✅ 使用构建后的优化文件
- ✅ 性能更好
- ✅ 模拟 Vercel 部署环境

**启动步骤：**

1. **先构建**（如果还没构建）：
   ```powershell
   npm run build:test-app
   ```

2. **启动生产服务器**：
   ```powershell
   cd test-app
   npm run start
   ```

**访问：**
- 打开浏览器访问：http://localhost:3000

## 🚀 快速启动

### 开发模式（最简单）


然后在浏览器中打开：http://localhost:3000

## 🔍 检查服务器是否运行

### 方法 1: 查看终端输出

启动后，终端会显示：
```
  ▲ Next.js 15.5.6
  - Local:        http://localhost:3000
  - Ready in 2.3s
```

### 方法 2: 检查端口

```powershell
# PowerShell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue
```

### 方法 3: 直接访问

在浏览器中打开 http://localhost:3000，如果页面加载，说明服务器正在运行。

## ⚠️ 常见问题

### 端口被占用

如果 3000 端口被占用：
- Next.js 会自动使用下一个可用端口（3001, 3002...）
- 查看终端输出确认实际端口
- 或者手动指定端口：`npm run dev -- -p 3001`

### 无法访问

1. **检查服务器是否启动**
   - 查看终端是否有错误信息
   - 确认端口号

2. **检查防火墙**
   - Windows 防火墙可能阻止了端口
   - 允许 Node.js 通过防火墙

3. **检查浏览器**
   - 尝试使用不同的浏览器
   - 清除浏览器缓存

### 页面显示错误

1. **检查终端错误信息**
2. **确认依赖已安装**：`cd test-app && npm install`
3. **重新构建**：`npm run build:test-app`
